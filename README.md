# Сервис бронирования пространств «Сириус. Аренда»

Бэкенд-приложение (REST API) для управления переговорными комнатами, учебными пространствами и бронированиями на федеральной территории «Сириус». Разработано в рамках конкурса «Всероссийский ИТ-раунд» по треку «Разработка и управление программным обеспечением».

## Технологический стек

- Язык программирования: Python 3.10+
- Фреймворк: FastAPI (асинхронный высокопроизводительный REST API)
- ORM / Валидация данных: SQLModel (библиотека на базе SQLAlchemy и Pydantic)
- База данных: SQLite (реляционная БД, данные хранятся локально в файле database.db)

---

## Инструкция по установке и запуску

Для проверки работоспособности выполните следующие шаги:

### 1. Подготовка рабочей директории

Клонируйте репозиторий с исходным кодом и перейдите в корень проекта:

git clone <url_репозитория>
cd sirius-arenda

### 2. Создание и активация виртуального окружения

Для операционной системы Windows:

python -m venv .venv
.venv\Scripts\activate

Для операционных систем macOS / Linux:

python -m venv .venv
source .venv/bin/activate

### 3. Установка зависимостей

Установите все необходимые внешние библиотеки, зафиксированные в requirements.txt:

pip install -r requirements.txt

### 4. Запуск сервера приложений

Запустите сервер FastAPI с помощью ASGI-сервера Uvicorn:

uvicorn main:app --reload

После успешного старта сервер начнет принимать запросы по адресу: http://127.0.0.1:8000

### 5. Запуск терминального клиента (опционально)

Для ручного интерактивного тестирования всех функций бэкенда через консоль, откройте второе окно терминала (не закрывая сервер) и запустите скрипт:

python cli.py

## Управление пространствами (Rooms)

POST /rooms — Создание нового пространства.

Тело запроса (JSON): Название комнаты (name), вместимость (capacity), список доступного оборудования (equipment).

GET /rooms — Получение списка всех зарегистрированных пространств.

GET /rooms/{id} — Получение подробной информации о конкретном пространстве по его уникальному ID.

PUT /rooms/{id} — Редактирование параметров существующего пространства.

DELETE /rooms/{id} — Полное удаление пространства из базы данных.

---

## Бронирование (Bookings)

POST /bookings — Создание бронирования пространства на указанный интервал времени.

Тело запроса (JSON): Идентификатор комнаты (room_id), время начала бронирования (start_time), время окончания бронирования (end_time), имя пользователя (user_name).

Бизнес-логика валидации: В приложении реализован алгоритм автоматической проверки наложений. Если выбранный временной интервал частично или полностью пересекается с уже существующим активным бронированием этой же комнаты, сервер блокирует операцию и возвращает HTTP-статус 409 Conflict с сообщением "Это время в комнате уже забронировано".

DELETE /bookings/{id} — Отмена бронирования. В соответствии с принципами сохранения истории, запись физически не удаляется, а переводится в статус "отменено".

GET /rooms/{id}/bookings — Получение актуального расписания активных бронирований для конкретной комнаты на выбранный день.

Параметры запроса (Query Params): date в формате YYYY-MM-DD (например, /rooms/1/bookings?date=2026-07-13). Возвращает упорядоченный список не отмененных бронирований.
